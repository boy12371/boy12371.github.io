Plugins

Kuangjia has a powerful plugin system, which makes it easy to extend functions without modifying the source code of the core module. There are two kinds of plugins in Kuangjia:

Script

If your plugin is relatively simple, it’s recommended to use a script. All you need to do is put your JavaScript files in the scripts folder and Kuangjia will load them during initialization.

Plugin

If your code is complicated or if you want to publish it to the NPM registry, we recommend using a plugin. First, create a folder in the node_modules folder. The name of this folder must begin with kuangjia- or Kuangjia will ignore it.

Your new folder must contain at least two files: one containing the actual JavaScript code and one package.json file that describes the purpose of the plugin and sets its dependencies.

You’ll also need to list your plugin as a dependency in the root package.json of your kuangjia instance in order for Kuangjia to detect and load it.

Tools

You can make use of the official tools provided by Kuangjia to accelerate development:

  • kuangjia-fs：File IO
  • kuangjia-util：Utilities
  • kuangjia-i18n：Localization (i18n)
  • kuangjia-pagination：Generate pagination data

Publishing

When your plugin is ready, you may consider publishing it to the plugin list to invite other people to start using it. Publishing your own plugins is very similar to updating documentation.

  1. Fork kuangjiajs/site

  2. Clone the repository to your computer and install dependencies.
